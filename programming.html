Development

HTML, or HyperText Markup Language, is a programming language used to display your basic mediums. What you’re reading right now is possible because of HTML. It’s the spine of the display in what you’re seeing now. By itself, it’s not the most attractive thing to look at. It has no styling options that are useful. That’s why HTML works best alongside other language models, or imports plug-ins from other areas (this will be talked more about in other sections).

The structure of a standard line of code in HTML is pretty simple. It contains an opening and closing tag, the attribute, and the content. All together, these create something called an Element.

CSS stands for Cascading Style Sheet. Its purpose is to provide decorative customization to a website. You can link the file to an HTML document and use identifiers to help, such as the type of opening and closing tags, or attributes of those elements such as a ‘class’ or ‘ID’. This is where more creative liberty can be taken to properly format things. Fonts, size, indentation, colors, just about anything can be customized to make your site more unique.

The syntax of a standard section of CSS contains the selector (what gets targeted in HTML is going to be changed), the property (whats being changed), and the property value (how it’s being changed). Any pair of a property value and a property is called a declaration. There can be as many declarations inside of a selector as the developer wishes.

JavaScript (JS), is one of the more difficult languages for front-end web development. Its purpose is to add interaction with a website. As far as learning this independently, it was more complicated than anything else. This deals with adding logic to a site. The biggest issue that I encountered was the strange syntax; it just never seemed to click how to make sense of it. A lot of times, you can write JavaScript code right in your HTML code using the 'script' tag. For my site in particular, there will be almost no examples of it in the code, and that was done intentionally. This site was meant to act like some kind of resume or showcase. User interaction wasn’t one of my biggest concerns as there weren’t many areas that would implement this effectively.
